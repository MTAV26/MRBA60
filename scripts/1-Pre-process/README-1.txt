README.txt
MRBA60 preprocessing workflow

Last updated: 2026-05-22

============================================================
1. PURPOSE
============================================================

This repository contains the preprocessing scripts used to build the main monthly input layers required by the MRBA60 burned-area harmonisation workflow.

The scripts generate or adjust the following datasets on a global 0.25-degree grid:

  - FireCCI51 burned area, 2003-2024
  - FireCCIS311 burned area, 2019-2024
  - MODIS MCD14ML active-fire count and FRP metrics, 2003-2024
  - ERA5 meteorological predictors, 2003-2024
  - MODIS MOD13C2 NDVI, 2003-2024
  - TerraClimate VPD, 2003-2024
  - GLEAM surface soil moisture, 2003-2024
  - ERA5-derived FWI exceedance metric, 2003-2024

The general objective is to convert heterogeneous raw products into temporally consistent, spatially aligned files suitable for subsequent modelling and validation steps.

============================================================
2. SCRIPT OVERVIEW
============================================================

A1_build_BA_Rdata.R
  Builds RData files for FireCCI51 and FireCCIS311 burned area products.
  It uses the function generar_rdata_fire() from:

    /mnt/disco6tb/MRBA60/scripts/0_functions/functions_fire_extract.R

  Main outputs:

    /mnt/disco6tb/MRBA60/data/A3_ADJ/longitude.RData
    /mnt/disco6tb/MRBA60/data/A3_ADJ/latitude.RData
    /mnt/disco6tb/MRBA60/data/A3_ADJ/FireCCI51_2003_2024_0.25degree.RData
    /mnt/disco6tb/MRBA60/data/A3_ADJ/FireCCIS311_2019_2024_0.25degree.RData

  It also checks array dimensions and plots global monthly burned area time series for FireCCI51 and FireCCIS311.

------------------------------------------------------------

B1_Count_and_FRP_025_Thr_and_Rat.R
  Processes raw MODIS MCD14ML active-fire text files.

  Input directory:

    /mnt/disco6tb/MRBA60/data/A1_RAW/MCD14ML

  Expected input format:

    MCD14ML.YYYYMM.006.03.txt.gz
    MCD14ML.YYYYMM.061.03.txt.gz

  Processing steps:

    1. Read monthly MCD14ML files.
    2. Keep only type == 0 fire detections.
    3. Compute scan angle from the sample column.
    4. Apply confidence and scan-angle filters.
    5. Rasterise detections to a global 0.25-degree grid.
    6. Save monthly GeoTIFF layers for:
         - FRP sum
         - FRP mean
         - FRP median
         - active-fire count

  Current filter settings:

    confidence >= 30
    absolute scan angle <= 30 degrees

  Output directory:

    /mnt/disco6tb/MRBA60/data/A2_TEMP/FRP-MCD14ML-TR

  Example outputs:

    MCD14ML_FRPsum_YYYYMM_025_conf30_angle30deg.tif
    MCD14ML_FRPmean_YYYYMM_025_conf30_angle30deg.tif
    MCD14ML_FRPmedian_YYYYMM_025_conf30_angle30deg.tif
    MCD14ML_AFcount_YYYYMM_025_conf30_angle30deg.tif

------------------------------------------------------------

B2_Join_Count_and_FRP_025_Thr_and_Rat.R
  Joins the monthly MODIS active-fire and FRP GeoTIFF files generated by B1 into 3D R arrays.

  Input directory:

    /mnt/disco6tb/MRBA60/data/A2_TEMP/FRP-MCD14ML-TR

  Output directory:

    /mnt/disco6tb/MRBA60/data/A3_ADJ

  Target period:

    200301-202412

  Variables processed:

    FRPsum
    FRPmean
    FRPmedian
    AFcount

  Current filter settings:

    conf30
    angle30

  Example outputs:

    MODIS-FRPsum_conf30_angle30-200301-202412-025.RData
    MODIS-FRPmean_conf30_angle30-200301-202412-025.RData
    MODIS-FRPmedian_conf30_angle30-200301-202412-025.RData
    MODIS-AFcount_conf30_angle30-200301-202412-025.RData

  The script transposes and flips the raster arrays so that they are stored consistently as lon x lat x time.

------------------------------------------------------------

C1_ERA5_adjustment.sh
  Adjusts ERA5 monthly predictors to the target 0.25-degree grid using CDO.

  Input directory:

    /mnt/disco6tb/MRBA60/data/A1_RAW

  Temporary directory:

    /mnt/disco6tb/MRBA60/data/A2_TEMP

  Output directory:

    /mnt/disco6tb/MRBA60/data/A3_ADJ

  Required target grid file:

    /mnt/disco6tb/MRBA60/data/A2_TEMP/grid_0.25.txt

  Variables and remapping methods:

    ERA5-TOT-PREC-2003-2024-MONTLY-025.nc    remapcon
    ERA5-TEMP-MEAN-2003-2024-MONTLY-025.nc   remapbil
    ERA5-WIND-SPEED-2003-2024-MONTLY-025.nc  remapbil
    ERA5-TOT-CLOUD-2003-2024-MONTLY-025.nc   remapbil

  Precipitation uses conservative remapping, while temperature, wind speed and cloud cover use bilinear interpolation.

------------------------------------------------------------

D1-convert_hdt_nc.sh
  Converts MOD13C2 HDF files to NetCDF by automatically identifying the NDVI subdataset.

  Input directory:

    /mnt/disco6tb/MHBA60/data/A1_RAW/MOD13C2_NDVI

  Output directory:

    /mnt/disco6tb/MHBA60/data/A2_TEMP/MOD13C2_NDVI

  Expected input format:

    MOD13C2.A*.hdf

  Output format:

    MOD13C2.A*.nc

  Note: this script uses the path root /mnt/disco6tb/MHBA60, while most other scripts use /mnt/disco6tb/MRBA60. Check whether this is intentional before running the workflow.

------------------------------------------------------------

D2_NDVI_025.R
  Aggregates monthly MOD13C2 NDVI NetCDF files from the native grid to the MRBA60 0.25-degree grid.

  Input directory:

    /mnt/disco6tb/MRBA60/data/A2_TEMP/MOD13C2_NDVI

  Output file:

    /mnt/disco6tb/MRBA60/data/A2_TEMP/MOD13C2_NDVI_2003_2024_025.nc

  Required coordinate files:

    /mnt/disco6tb/MRBA60/data/A2_TEMP/longitude.RData
    /mnt/disco6tb/MRBA60/data/A2_TEMP/latitude.RData

  Processing details:

    - Period: 2003-2024
    - Scale factor applied: 0.0001
    - Missing value: 1e20
    - Default aggregation: area-weighted mean
    - Output convention: CF-1.8
    - Output variable: NDVI(lon, lat, time)
    - Time units: days since 1970-01-01 00:00:00

  The script validates input geometry, resolution, extent and CRS before processing each monthly layer.

------------------------------------------------------------

E1_regrid_VPD_0.25deg.sh
  Regrids TerraClimate VPD files to 0.25 degrees using bilinear interpolation and merges annual files into a single time series.

  Input directory:

    /mnt/disco6tb/MHBA60/data/A1_RAW/VPD_v1.1

  Temporary/output directories:

    /mnt/disco6tb/MHBA60/data/A2_TEMP
    /mnt/disco6tb/MHBA60/data/A2_TEMP/VPD

  Required target grid file:

    /mnt/disco6tb/MHBA60/data/A2_TEMP/grid_0.25.txt

  Expected input format:

    TerraClimate_vpd_YYYY.nc

  Main merged output:

    /mnt/disco6tb/MHBA60/data/A2_TEMP/vpd_2003-2024_0.25deg_bil.nc

  Note: this script uses the path root /mnt/disco6tb/MHBA60. Check consistency with the MRBA60 directory structure before running.

------------------------------------------------------------

F1-SoilMosture_CDO.sh
  Merges and regrids GLEAM v4.2b surface soil moisture to 0.25 degrees using CDO.

  Input directory:

    /mnt/disco6tb/MHBA60/data/A1_RAW/SMs_v4.2b

  Output directory:

    /mnt/disco6tb/MHBA60/data/A2_TEMP

  Required target grid file:

    /mnt/disco6tb/MHBA60/data/A2_TEMP/grid_0.25.txt

  Intermediate outputs:

    filelist_SMs_GLEAM_v4.2b.txt
    SMs_2003_2024_GLEAM_v4.2b_MO.nc
    srcgrid_GLEAM_0.1deg.txt

  Final output:

    SMs_2003_2024_GLEAM_v4.2b_MO_025deg_bil.nc

  The script defines the native GLEAM 0.1-degree grid explicitly before applying bilinear remapping.

  Note: this script uses the path root /mnt/disco6tb/MHBA60. Check consistency with the MRBA60 directory structure before running.

------------------------------------------------------------

G1-FWI_CDO.sh
  Builds a monthly FWI-derived predictor from daily ERA5 FWI files.

  Input directory:

    /mnt/disco6tb/MHBA60/data/A1_RAW/FWI_v4.1

  Output directory:

    /mnt/disco6tb/MHBA60/data/A2_TEMP/FWI

  Required target grid file:

    /mnt/disco6tb/MHBA60/data/A2_TEMP/grid_0.25.txt

  Expected input format:

    fwi-era5-YYYYMMDD.nc

  Period:

    2003-2024

  Processing steps:

    1. Concatenate daily files by calendar month across all years.
    2. Remap monthly daily stacks to 0.25 degrees using bilinear interpolation.
    3. Compute the monthly calendar-specific P95 threshold.
    4. Count, for each year-month, the number of days with FWI > P95.
    5. Merge all monthly counts into a complete 2003-2024 time series.

  Final output:

    fwi-era5_count_exceed95_200301-202412_0.25.nc

  This final file contains the monthly number of days exceeding the calendar-month 95th percentile of FWI.

  Note: this script uses the path root /mnt/disco6tb/MHBA60. Check consistency with the MRBA60 directory structure before running.

============================================================
3. RECOMMENDED EXECUTION ORDER
============================================================

Run the scripts in the following order:

  1. A1_build_BA_Rdata.R
  2. B1_Count_and_FRP_025_Thr_and_Rat.R
  3. B2_Join_Count_and_FRP_025_Thr_and_Rat.R
  4. C1_ERA5_adjustment.sh
  5. D1-convert_hdt_nc.sh
  6. D2_NDVI_025.R
  7. E1_regrid_VPD_0.25deg.sh
  8. F1-SoilMosture_CDO.sh
  9. G1-FWI_CDO.sh

Some scripts are independent once the target grid file is available, but this order follows the logical structure of the preprocessing chain.

============================================================
5. DIRECTORY STRUCTURE
============================================================

The workflow uses the following general structure:

  A1_RAW
    Raw input products.

  A2_TEMP
    Temporary and intermediate files.

  A3_ADJ
    Adjusted, harmonised or model-ready files.

Main MRBA60 root used by most scripts:

  /mnt/disco6tb/MRBA60

Additional root appearing in several scripts:

  /mnt/disco6tb/MHBA60

Important consistency note:

  Some auxiliary scripts currently point to /mnt/disco6tb/MHBA60 instead of /mnt/disco6tb/MRBA60. Before executing the full workflow, verify whether MHBA60 is an intended directory or whether those paths should be updated to MRBA60.

============================================================
6. TARGET GRID
============================================================

The common target grid is a global lon-lat grid at 0.25-degree resolution:

  - ncols: 1440
  - nrows: 720
  - xmin: -180
  - xmax: 180
  - ymin: -90
  - ymax: 90
  - CRS: EPSG:4326

Several scripts require the CDO grid definition file:

  grid_0.25.txt

Depending on the script, this file is expected under:

  /mnt/disco6tb/MRBA60/data/A2_TEMP/grid_0.25.txt

or:

  /mnt/disco6tb/MHBA60/data/A2_TEMP/grid_0.25.txt

============================================================
8. QUALITY-CONTROL CHECKS INCLUDED
============================================================

The scripts include several internal checks:

  - Check whether required input files exist.
  - Check whether CDO is available.
  - Check whether the target grid file exists.
  - Check monthly file availability.
  - Warn when months are missing from the expected 2003-2024 sequence.
  - Validate raster dimensions, resolution, extent and CRS.
  - Confirm that output arrays have the expected dimensionality.
  - Print or plot basic time-series diagnostics.

Recommended additional checks after running the workflow:

  - Confirm that all final files cover the expected 264 monthly time steps for 2003-2024.
  - Confirm that lon and lat vectors are consistent across all RData and NetCDF outputs.
  - Check that units are correctly documented in each NetCDF file.
  - Inspect global monthly time series for unrealistic discontinuities.
  - Confirm that all auxiliary predictors use the same land/ocean convention before modelling.

============================================================
9. KNOWN POINTS TO REVIEW BEFORE EXECUTION
============================================================

1. Directory root inconsistency

   Several scripts use /mnt/disco6tb/MHBA60, while the main burned-area scripts use /mnt/disco6tb/MRBA60. Verify whether this is intentional.

2. Typographical issue in file names

   The ERA5 file names use MONTLY instead of MONTHLY. This is preserved because the scripts expect that exact spelling.

3. NDVI coordinate files

   D2_NDVI_025.R expects longitude.RData and latitude.RData under:

     /mnt/disco6tb/MRBA60/data/A2_TEMP

   However, A1_build_BA_Rdata.R writes these files under:

     /mnt/disco6tb/MRBA60/data/A3_ADJ

   Check whether copies exist in A2_TEMP or whether D2_NDVI_025.R should point to A3_ADJ.

4. FireCCI extraction function

   A1_build_BA_Rdata.R depends on:

     /mnt/disco6tb/MRBA60/scripts/0_functions/functions_fire_extract.R

   This file must be available before running the burned-area preprocessing.

5. MCD14ML filters

   The current active-fire preprocessing uses confidence >= 30 and scan angle <= 30 degrees. If other threshold combinations are needed, modify conf_levels and angle_levels_deg in B1 and the corresponding conf_labels and angle_levels in B2.

12. CONTACT / MAINTENANCE NOTES
============================================================

Before sharing or archiving this workflow, it is recommended to:

  - Replace machine-specific absolute paths with configurable variables.
  - Add a master configuration file for root directories and periods.
  - Standardise MRBA60 versus MHBA60 directory names.
  - Store the grid_0.25.txt file together with the scripts or document how it was generated.
  - Record software versions for R, terra, ncdf4, CDO and GDAL.
  - Add checksums or file manifests for raw input products.
