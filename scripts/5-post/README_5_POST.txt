README - 5-post
===============

This folder contains the post-processing scripts applied after the biome-level random forest harmonisation step. These scripts assemble the biome outputs into global arrays, restore historical extreme burned-area events that were not adequately reproduced by the model, and apply a final empirical quantile mapping correction.

The target product is the monthly MRBA60 burned area grid for 2003-2024 at 0.25-degree spatial resolution.


Scripts included
================

  5-Harmonisised_Unir_biomas.R

  6-ReintroducirExtremNovistos.R

  7-HBA-QM001M-6.R


General workflow
================

The post-processing workflow is composed of three sequential steps:

  1. Merge biome-level harmonised burned area outputs into global arrays.

  2. Reintroduce historical FireCCI51 extreme events that were not represented by the harmonised model, using biome-month thresholds derived from the common calibration period.

  3. Apply empirical quantile mapping by biome and month to improve the distributional agreement between the harmonised product and FireCCIS311 during the common period.

The expected execution order is therefore:

  1. 5-Harmonisised_Unir_biomas.R
  2. 6-ReintroducirExtremNovistos.R
  3. 7-HBA-QM001M-6.R

Required previous processing blocks
===================================

Before running the scripts in this folder, the following previous steps must have been completed:

  1. FireCCI51 and FireCCIS311 monthly burned area RData files must exist.

  2. FireMask_AF3030F must have been generated.

  3. The 4-model block must have produced biome-level harmonised outputs, including:

       BA_<Modelo>_<biome>_FireHarmonized_Common.RData
       BA_<Modelo>_<biome>_FireHarmonized_Full.RData

  4. The 4-model block must have generated the monthly biome threshold files:

       Max_Tope_COMMON_ByMonth_<biome>.csv

  5. The biome shapefile must be available:

       continental-biomes_dinerstein_V10.shp


Common input datasets
=====================

The scripts use the following common inputs:

  longitude.RData
  latitude.RData
  continental-biomes_dinerstein_V10.shp
  FireCCI51_2003_2024_0.25degree.RData
  FireCCIS311_2019_2024_0.25degree.RData
  FireMask_AF3030F.RData

The burned area variables are converted from square metres to square kilometres by dividing by 1e6.


5-Harmonisised_Unir_biomas.R
============================

Purpose
-------

This script reconstructs global harmonised burned area arrays by merging the biome-level outputs generated by the random forest harmonisation step.

Main operations
---------------

  1. Load longitude, latitude, FireCCI51, FireCCIS311, FireMask_AF3030F and the biome shapefile.

  2. Expand FireCCIS311 from the common period 2019-2024 into a full 2003-2024 array, using NA outside the common period.

  3. Build the latitude-dependent grid-cell area matrix.

  4. Loop through all biomes in the biome shapefile.

  5. For each biome, reconstruct the grid-cell assignment using the biome polygons and reassign active cells without a biome to the nearest valid biome.

  6. Load the biome-level harmonised common-period and full-period RData files.

  7. Insert each biome-level harmonised array into the corresponding position of the global array.

  8. Save the global harmonised arrays.

Main outputs
------------

  /mnt/disco6tb/MRBA60-2/results/B1-MRBA60-2003-2024/RData/BA_B1-MRBA60-2003-2024global_BA_FireHarmonized_Common.RData

  /mnt/disco6tb/MRBA60-2/results/B1-MRBA60-2003-2024/RData/BA_B1-MRBA60-2003-2024global_BA_FireHarmonized_Full.RData

Notes
-----

The script uses the FireMask_AF3030F mask to restrict the merging process to cells belonging to the valid fire domain. It also applies the biome reassignment logic already used in previous scripts, so that grid cells with fire presence but no direct biome polygon assignment are assigned to the nearest valid biome.


6-ReintroducirExtremNovistos.R
==============================

Purpose
-------

This script identifies and restores historical FireCCI51 burned area events that exceed biome-month thresholds derived from the common period and that may not have been fully represented by the harmonised model.

This step is designed to preserve historical extreme events, especially during 2003-2018, when FireCCIS311 is not available.

Main operations
---------------

  1. Load FireCCI51, FireCCIS311 and the global harmonised burned area product.

  2. Build a combined status mask indicating cells/months where any of the three products has burned area information.

  3. Load the biome shapefile and loop over all biomes.

  4. For each biome, reconstruct the spatial biome mask.

  5. Read the monthly threshold file for the biome:

       Max_Tope_COMMON_ByMonth_<biome>.csv

     The threshold is based on common-period FireCCI51 and/or FireCCIS311 maxima, depending on the configured column.

  6. Identify FireCCI51 values above the monthly threshold.

  7. Store exceedance masks and values.

  8. Correct the harmonised product by replacing selected historical extreme values with FireCCI51 values where the threshold rule is met.

Main expected outputs
---------------------

The script creates an output folder for threshold exceedance diagnostics:

  /mnt/disco6tb/MRBA60-2/results/B1-MRBA60-2003-2024/RData/threshold_exceedance_F51

The script also generates a corrected harmonised product, used by the following quantile mapping script:

  BA_harmonised_correctedByF51Tope_B1-MRBA60-2003-2024.RData

Notes
-----

This script is a post-processing safeguard. It prevents the final harmonised product from suppressing exceptional historical FireCCI51 events that are outside the modelled range observed during the common FireCCIS311 period.

The script is configured to search for threshold files with the pattern:

  Max_Tope_COMMON_ByMonth_*.csv
  Max_Tope_COMMON_ByMonth_*.xlsx

The exact threshold column should be checked before execution. The script contains logic for reading monthly topes and matching them to each biome.


7-HBA-QM001M-6.R
================

Purpose
-------

This script applies empirical quantile mapping (EQM) to the harmonised burned area product. The correction is performed using positive burned area values only, by biome and month.

The aim is to improve the distributional agreement between the harmonised product and FireCCIS311 during the common period 2019-2024, while preserving physically plausible values.

Main operations
---------------

  1. Load FireCCI51, FireCCIS311 and the harmonised product corrected by the FireCCI51 threshold step.

  2. Build a common information mask where any of the products has burned area.

  3. Apply a minimum physical threshold to the input harmonised burned area.

  4. Compute latitude-dependent minimum and maximum burned area constraints.

  5. Loop over biomes.

  6. For each biome and month, fit empirical quantile mapping using positive values only.

  7. Use a three-month moving window for training.

  8. Require a minimum number of positive cells and valid years before applying QM.

  9. Apply the fitted QM correction to the full 2003-2024 period.

  10. Apply minimum and maximum physical constraints after correction.

  11. Restore FireCCI51 values above the threshold for the historical period where required.

  12. Generate audit tables, Q-Q plots, histograms and scatter plots.

Key parameters
--------------

  slice_train = 193:264

    Training period corresponding to 2019-2024.

  thr_min = 0

    No additional post-QM threshold beyond the physical minimum rule.

  min_cells_qm = 30

    Minimum number of positive cells per year required to activate QM.

  min_years_pos = 2

    Minimum number of valid years required for QM.

  amin_px_eq = 0.09

    Minimum physical burned area threshold at the equator, in km2.

  qstep = 0.01

    Quantile step used by fitQmapQUANT.

Main outputs
------------

The script writes final corrected outputs and diagnostics under:

  /mnt/disco6tb/MRBA60-2/results/B1-MRBA60-2003-2024/RData
  /mnt/disco6tb/MRBA60-2/results/B1-MRBA60-2003-2024/csv
  /mnt/disco6tb/MRBA60-2/results/B1-MRBA60-2003-2024/plot_QM001M
  /mnt/disco6tb/MRBA60-2/results/B1-MRBA60-2003-2024/plot/qqplots_eqm

The main final array is the quantile-mapped global harmonised burned area product for 2003-2024.

Notes
-----

The quantile mapping is fitted only on positive burned area values. Zeros and missing values are handled through the common information mask and the physical filtering steps.

For biomes without enough positive FireCCIS311 data in the common period, the script copies the harmonised input after applying physical constraints, rather than forcing an unstable QM correction.

The script applies both minimum and maximum physical constraints. The maximum is based on the estimated 0.25-degree grid-cell area. The minimum is latitude-adjusted from amin_px_eq.


Physical constraints
====================

The post-processing block applies two main physical controls:

  Minimum burned area threshold:

    Values greater than zero but below the latitude-adjusted minimum threshold are set to zero.

  Maximum burned area threshold:

    Values larger than the estimated grid-cell area are capped to the cell area.

These constraints are especially important after random forest prediction and quantile mapping, because both procedures can generate values outside the physically meaningful range.


Biome handling
==============

All scripts use the continental biome shapefile and the field:

  cont_bm

The scripts reconstruct biome masks using spatial intersections between grid-cell centres and biome polygons. Cells with fire presence but no direct biome assignment can be reassigned to the nearest biome with a valid polygon assignment. This keeps coastal or boundary fire pixels within the modelling domain.


Temporal periods
================

Full period:

  2003-01 to 2024-12

Common period:

  2019-01 to 2024-12

Historical period without FireCCIS311:

  2003-01 to 2018-12

The quantile mapping calibration period is the common period, indexed as:

  193:264

Recommended checks after execution
==================================

After running 5-Harmonisised_Unir_biomas.R:

  Check that the global common and full RData files exist.
  Plot several monthly layers to confirm that the global mosaic is spatially complete.
  Confirm that dimensions match [lon, lat, time] = [1440, 720, 264].

After running 6-ReintroducirExtremNovistos.R:

  Check the threshold_exceedance_F51 folder.
  Review how many cells/months were corrected by the FireCCI51 threshold rule.
  Confirm that the corrected harmonised RData file exists.

After running 7-HBA-QM001M-6.R:

  Review the QM audit CSV files.
  Check Q-Q plots, histograms and scatter plots.
  Confirm that no burned area value exceeds the grid-cell area.
  Confirm that very small positive values below the minimum threshold were removed.
  Compare global and biome-level time series before and after QM.


Important notes
===============

The post-processing block should be run only after the full 4-model harmonisation block has completed successfully.

The reintroduction of extremes is not a general replacement of the harmonised product by FireCCI51. It is a targeted correction for events exceeding biome-month thresholds.

The quantile mapping step is trained against FireCCIS311 during 2019-2024 and then applied to the full 2003-2024 series.

The final product should be interpreted as the harmonised random forest product after physical filtering, historical extreme-event restoration and distributional correction.
